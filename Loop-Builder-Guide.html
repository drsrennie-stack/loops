Building a New Loop Yourself

How to make a loop with the Loop Builder and publish it, without writing any code.

1What the builder does

The Loop Builder is the file loop-builder.html. Double-click it to open it in your browser. It runs entirely on your computer; nothing is uploaded anywhere. You give it slides and answer keys, and it hands you a finished loop as a ZIP. Your job after that is one push to GitHub.

At the top it asks what you want to do:

Make a new loopCreates a fresh card on the home page.
Add cards to an existing loopTacks new stations onto a loop you already posted, even the built-in ones. Pick the loop, add only the new slides, and it numbers them to continue that loop.
For "add cards" mode: load your current custom-loops.js in step 2 first, so your earlier work is kept and your own loops show up in the list to choose from.

2What your slides need to look like

Each station is two slides: the question (arrows pointing at the structures, no answers), then the answer (same picture with the labels shown). That is exactly how your loop decks already work.

You can feed the builder slides two ways:

Upload imagesDrag in your slide images in order: question, answer, question, answer. Name them so they sort right (01, 02, 03…).
Import a PDFDrag in a PowerPoint or Keynote saved as PDF. Then tag each page Q, A, or Skip (skip title slides and blank animation frames). The tool pairs each Q with the next A.

3The five steps in the tool

  1. Type a loop name (the id and folder name fill in automatically). Pick an icon and write a short description.
  2. Choose Upload images or Import a PDF and load your slides. For a PDF, tag every page Q, A, or Skip.
  3. Click Build the pairs. Each station appears with its question and answer picture. Type the answer key in the box (both structures on one line is fine, like "Iliac Crest; Acetabulum").
  4. Only if you have built loops before: drop your current custom-loops.js in step 4 so the new loop is added to them, not replacing them. First time, skip this.
  5. Click Build & download loop ZIP. A file named loop-<id>.zip lands in your Downloads.

4Publishing it (one push)

Unzip loop-<id>.zip. Inside you get an images folder (with your loop inside it) and a custom-loops.js file. In your loops repo on your computer:

loops/ ├─ index.html ├─ custom-loops.js ← replace with the new one from the ZIP └─ images/ └─ <your-loop-id>/ ← drop this new folder in here

Then open GitHub Desktop, write a short summary, Commit to main, and Push origin. (Same routine as your GitHub desk sheet.) Your new loop card shows up on the home page automatically, and on your course page.

Why it just appears: your app reads custom-loops.js every time it loads and adds whatever loops are listed there. You never edit index.html.

5If something looks off

Pairs look mismatchedYour slides were out of order, or a PDF page was tagged wrong. Re-tag and click Build the pairs again.
A card is blankThe images/<id> folder did not land inside images/, or its id does not match custom-loops.js. Check the folder name.
The loop did not appearcustom-loops.js was not pushed, or you reused an id already in use. Give it a new name and rebuild.
PDF will not loadRe-save it as a standard PDF. Very large decks take a minute, give it time.
